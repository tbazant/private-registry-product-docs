[#pr-configure-rancher-oidc]
== Configurar {ranchera} como un Proveedor de Identidad OIDC

:revdate: 2026-05-20
:page-revdate: {revdate}

Esta guía explica cómo configurar {ranchera} para actuar como un Proveedor de Identidad OIDC, permitiendo a los usuarios autenticarse en aplicaciones externas como {productname} utilizando sus credenciales de {ranchera}.


[#pr-configure-rancher-oidc-enable-flag]
=== Paso 1: Habilitar la bandera de característica oidc-provider

. Inicia sesión en la interfaz de usuario de {ranchera} como administrador.
. Haz clic en el menú de tres líneas (☰) en la parte superior izquierda y ve a menu:Configuración Global[Banderas de características].
. Encuentra la bandera `oidc-provider`, haz clic en el icono de *Más Acciones* (⋮) y haz clic en *Activar*.

image::activate-oidc-rancher.png[Configurar {ranchera} como un Proveedor de Identidad OIDC, width=100%]


[#pr-configure-rancher-oidc-create-client]
=== Paso 2: Crear un recurso OIDCClient

{ranchera} utiliza un recurso personalizado `OIDCClient` para registrar aplicaciones en sentido descendente.

. Crea un archivo llamado `rancher-oidc-client.yaml` con el siguiente contenido:
+
[source,yaml]
----
apiVersion: management.cattle.io/v3
kind: OIDCClient
metadata:
  name: spr-client
spec:
  tokenExpirationSeconds: 600
  refreshTokenExpirationSeconds: 3600
  redirectURIs:
    # Replace this with the actual callback URL of your SUSE private registry instance
    - "https://<SUSE_PRIVATE_REGISTR_URL>/c/oidc/callback"
----

. Aplica el archivo al clúster donde se está ejecutando {ranchera}:
+
[source,bash,subs="+attributes"]
----
{prompt_user}kubectl apply -f rancher-oidc-client.yaml
----

[#pr-configure-rancher-oidc-retrieve-credentials]
=== Paso 3: Recupera el ID de Cliente y el Secreto

Una vez que se crea el recurso, {ranchera} completa automáticamente el `clientID` y provisiona un Secreto {kube} que contiene el `clientSecret`.

. Obtén el ID de Cliente generado:
+
[source,bash,subs="+attributes"]
----
{prompt_user}kubectl get oidcclient spr-client -o jsonpath="{.status.clientID}"
----

. Obtén el Secreto de Cliente. Recuerda reemplazar `<YOUR_CLIENT_ID>` con el ID recuperado en el paso anterior:
+
[source,bash,subs="+attributes"]
----
{prompt_user}kubectl get secret <YOUR_CLIENT_ID>
  -n cattle-oidc-client-secrets
  -o jsonpath="{.data.client-secret-1}" | base64 -d
----

[#pr-configure-rancher-oidc-configure-spr]
=== Paso 4: Configurar {productname}

Puedes configurar {productname} para usar {ranchera} como su proveedor OIDC pasando los valores a través de {helm}.
Usa el bloque `core.configureUserSettings` en tu configuración de `values-oidc.yaml`.

Lo siguiente es un bloque de ejemplo utilizando tu endpoint de {ranchera} y las credenciales recuperadas arriba.
Reemplaza los valores de `<YOUR_CLIENT_ID>` y `<YOUR_CLIENT_SECRET>`.

[source,yaml]
----
core:
  configureUserSettings: |
    {
      "auth_mode": "oidc_auth",
      "oidc_name": "Rancher",
      "oidc_endpoint": "<RANCHER_URL>/oidc",
      "oidc_client_id": "<YOUR_CLIENT_ID>",
      "oidc_client_secret": "<YOUR_CLIENT_SECRET>",
      "oidc_scope": "openid,profile,offline_access",
      "oidc_verify_cert": false,
      "oidc_auto_onboard": true,
      "oidc_user_claim": "preferred_username",
      "oidc_groups_claim": "groups",
      "oidc_admin_group": "spr-admins"
    }
----

[NOTE]
====
Asegúrate de que `oidc_verify_cert` esté configurado en `false` si tu instancia de {ranchera} está utilizando certificados autofirmados.
Al especificar `oidc_admin_group`, cualquier usuario {ranchera} que pertenezca al grupo `spr-admins` recibirá automáticamente privilegios de administrador del sistema en {productname}.
====
