[#pr-configure-rancher-oidc]
== Configurez {ranchera} en tant que fournisseur d'identité OIDC

:revdate: 2026-05-20
:page-revdate: {revdate}

Ce guide explique comment configurer {ranchera} pour agir en tant que fournisseur d'identité OIDC, permettant aux utilisateurs de s'authentifier dans des applications externes telles que {productname} en utilisant leurs identifiants {ranchera}.


[#pr-configure-rancher-oidc-enable-flag]
=== Étape 1 : Activez le drapeau de fonctionnalité oidc-provider

. Connectez-vous à l'interface utilisateur de {ranchera} en tant qu'administrateur.
. Cliquez sur le menu trois lignes (☰) en haut à gauche et allez à menu:Paramètres globaux[Drapeaux de fonctionnalités].
. Trouvez le drapeau `oidc-provider`, cliquez sur l'icône *Plus d'actions* (⋮), puis cliquez sur *Activer*.

image::activate-oidc-rancher.png[Configurer {ranchera} en tant que fournisseur d'identité OIDC, width=100%]


[#pr-configure-rancher-oidc-create-client]
=== Étape 2 : Créez une ressource OIDCClient

{ranchera} utilise une ressource personnalisée `OIDCClient` pour enregistrer les applications en aval.

. Créez un fichier nommé `rancher-oidc-client.yaml` avec le contenu suivant{nbsp}:
+
[source,yaml]
----
apiVersion: management.cattle.io/v3
kind: OIDCClient
metadata:
  name: spr-client
spec:
  tokenExpirationSeconds: 600
  refreshTokenExpirationSeconds: 3600
  redirectURIs:
    # Replace this with the actual callback URL of your SUSE private registry instance
    - "https://<SUSE_PRIVATE_REGISTR_URL>/c/oidc/callback"
----

. Appliquez le fichier au cluster où {ranchera} est en cours d'exécution{nbsp}:
+
[source,bash,subs="+attributes"]
----
{prompt_user}kubectl apply -f rancher-oidc-client.yaml
----

[#pr-configure-rancher-oidc-retrieve-credentials]
=== Étape 3{nbsp}: Récupérez l'ID client et le secret

Une fois la ressource créée, {ranchera} remplit automatiquement le `clientID` et provisionne un secret {kube} contenant le `clientSecret`.

. Obtenez l'ID client généré{nbsp}:
+
[source,bash,subs="+attributes"]
----
{prompt_user}kubectl get oidcclient spr-client -o jsonpath="{.status.clientID}"
----

. Récupérez le secret client. N'oubliez pas de remplacer `<YOUR_CLIENT_ID>` par l'ID récupéré à l'étape précédente{nbsp}:
+
[source,bash,subs="+attributes"]
----
{prompt_user}kubectl get secret <YOUR_CLIENT_ID>
  -n cattle-oidc-client-secrets
  -o jsonpath="{.data.client-secret-1}" | base64 -d
----

[#pr-configure-rancher-oidc-configure-spr]
=== Étape 4{nbsp}: Configurez {productname}

Vous pouvez configurer {productname} pour utiliser {ranchera} comme fournisseur OIDC en passant les valeurs via {helm}.
Utilisez le bloc `core.configureUserSettings` dans votre configuration `values-oidc.yaml`.

L'exemple suivant est un bloc utilisant votre {ranchera} point de terminaison et les identifiants récupérés ci-dessus.
Remplacez les valeurs de `<YOUR_CLIENT_ID>` et `<YOUR_CLIENT_SECRET>`.

[source,yaml]
----
core:
  configureUserSettings: |
    {
      "auth_mode": "oidc_auth",
      "oidc_name": "Rancher",
      "oidc_endpoint": "<RANCHER_URL>/oidc",
      "oidc_client_id": "<YOUR_CLIENT_ID>",
      "oidc_client_secret": "<YOUR_CLIENT_SECRET>",
      "oidc_scope": "openid,profile,offline_access",
      "oidc_verify_cert": false,
      "oidc_auto_onboard": true,
      "oidc_user_claim": "preferred_username",
      "oidc_groups_claim": "groups",
      "oidc_admin_group": "spr-admins"
    }
----

[NOTE]
====
Assurez-vous que `oidc_verify_cert` est défini sur `false` si votre instance {ranchera} utilise des certificats auto-signés.
En spécifiant `oidc_admin_group`, tout utilisateur {ranchera} appartenant au groupe `spr-admins` se verra automatiquement octroyer des privilèges d'administrateur système dans {productname}.
====
