[#pr-helm-chart-override]
== 覆盖 {productname} {helm} 图表
:revdate: 2025-10-15
:page-revdate: {revdate}

{productname} ({productnameshort}) {helm} 图表以默认值交付。
您可以通过以下方式之一调整 {helm} 图表安装：

* 在 `--set` 命令行的 `helm install` 标志后附加特定参数，例如：
[source]
$ helm install <RELEASE_NAME> \
oci://registry.suse.com/private-registry/private-registry-helm \
--namespace <PRIVATE_REGISTRY_NAMESPACE> \
--set harborAdminPassword=<MY_PASSWORD> \
--set externalURL=https://<PRIVATE_REGISTRY_FQDN> \
--set expose.ingress.hosts.core=<PRIVATE_REGISTRY_FQDN>
* Create a {suse} custom `suse_registry_override.yaml` file and pass it to the `--f` flag, for example:
[source]
  $ helm install <RELEASE_NAME> \
  oci://registry.suse.com/private-registry/private-registry-helm \
  --namespace <PRIVATE_REGISTRY_NAMESPACE>
  -f suse_registry_override.yaml

[#pr-helm-chart-override-examples]
=== {sregistry} {helm} 覆盖文件示例
.使用 {ingress} 的最小部署
[example]
----
expose:
  type: ingress //<1>
  ingress:
    hosts:
      core: <PRIVATE_REGISTRY_FQDN> //<2>

externalURL: https://<PRIVATE_REGISTRY_FQDN> //<3>

harborAdminPassword: "<MY_PASSWORD>" //<4>

database:
  internal:
    password: "<MY_PASSWORD_POSTGRESQL>"

redis:
  internal:
   password: "<MY_PASSWORD_REDIS>"
----
[callouts]
<1> {sregistry} 是如何暴露的。可以是 `ingress`、`loadBalancer`、`nodePort` 或 `clusterIPhis`。默认值为 `ingress`。
<2> {kube} 内部网络配置的主机名。
<3> {sregistry} 应用程序运行的 URL。它用于生成用户界面中的链接、重定向以及 API 响应。
<4> 应用程序的管理员密码。

.使用 `loadBalancer` 的典型部署
[example]
----
expose:
  type: loadBalancer //<1>
  tls:
    enabled: true
    certSource: secret //<2>
    secret:
      secretName: <SECRET_NAME>

    auto:
      commonName: <PRIVATE_REGISTRY_FQDN> //<3>

externalURL: https://<PRIVATE_REGISTRY_FQDN> //<4>

harborAdminPassword: "<MY_PASSWORD>" //<5>

database:
  internal:
    password: "<MY_PASSWORD_POSTGRESQL>"

redis:
  internal:
   password: "<MY_PASSWORD_REDIS>"
----
[callouts]
<1> {sregistry} 是如何暴露的。可以是 `ingress`、`loadBalancer`、`nodePort` 或 `clusterIP`。默认值为 `ingress`。
<2> 可以是 `auto`、`secret` 或 `none`。根据选项，您可能需要包含额外的值。
<3> 使用 TLS 加密时，此字段必须与 `externalURL` 值匹配。
<4> {sregistry} 应用程序运行的 URL。它用于生成用户界面中的链接、重定向以及 API 响应。
<5> 应用程序的管理员密码。

[#pr-helm-chart-override-parameters]
=== 覆盖 {helm} 图表参数和值

以下表格列出了所有参数及其描述，您可以使用这些参数来覆盖默认安装值。

.全局参数
`global.imageRegistry`:: 为所有镜像设置容器镜像注册表的全局覆盖。
`global.imagePullSecrets`:: 为访问容器镜像注册表设置全局拉取密钥。

.通用参数
`harborAdminPassword`:: 为 {harbor} 管理员设置初始密码。在部署后从门户更改它。默认值为 `Harbor12345`。
`externalURL`:: 指定 `harbor-core` 服务的外部 URL。默认值为 `\https://core.harbor.domain`。
`existingSecretAdminPasswordKey`:: 设置包含 {harbor} 管理员密码的密钥名称。默认值为 `HARBOR_ADMIN_PASSWORD`。
`imagePullSecrets`:: 为所有部署设置 `imagePullSecrets` 名称。
`updateStrategy.type`:: 为具有持久卷的部署设置更新策略。接受 `RollingUpdate` 或 `Recreate`。当卷不支持 RWM 时使用 `Recreate`。默认值为 `RollingUpdate`。
`logLevel`:: 为 {harbor} 服务设置日志级别。接受 `fatal`、`error`、`warn`、`info`、`debug` 或 `trace`。默认值为 `debug`。
`enableMigratehelmHook`:: 通过 {helm} 钩子运行数据库迁移作业。当 `true` 时，将迁移作业与 `harbor-core` 分开。默认值为 `false`。
`caSecretName`:: 指定包含 `ca.crt` 密钥的秘密名称。

.代理参数
`proxy.httpProxy`:: 指定HTTP代理服务器的URL。默认值为 `""`。
`proxy.httpsProxy`:: 指定HTTPS代理服务器的URL。默认值为 `""`。
`proxy.noProxy`:: 设置绕过代理配置的URL。默认值为 `127.0.0.1,localhost,.local,.internal`。
`proxy.components`:: 设置使用代理配置的组件。默认值为 `["core","jobservice","trivy"]`。

.暴露相关参数
`expose.type`:: 指定服务暴露类型：`ingress`、`clusterIP`、`nodePort`或`loadBalancer`。默认值为 `ingress`。
`expose.tls.enabled`:: 启用TLS。默认值为 `true`。
`expose.tls.certSource`:: 将TLS证书源设置为`auto`、`secret`或`none`。默认值为 `auto`。
`expose.tls.auto.commonName`:: 当类型不是`ingress`时，设置证书的通用名称。
`expose.tls.secret.secretName`:: 指定包含`tls.crt`（证书）和`tls.key`（私钥）的秘密名称。
`expose.ingress.hosts.core`:: 在{harbor}规则中设置{ingress}内核服务主机。默认值为 `core.harbor.domain`。
`expose.ingress.controller`:: 设置{ingress}控制器类型。支持`default`、`gce`、`alb`、`f5-bigip`和`ncp`。默认值为 `default`。
`expose.ingress.kubeVersionOverride`:: 为{kube}模板覆盖{ingress}版本。
`expose.ingress.annotations`:: 设置{ingress}注释。
`expose.ingress.labels`:: 设置{ingress}特定标签。默认值为 `{}`。
`expose.clusterIP.name`:: 设置ClusterIP服务名称。默认值为 `harbor`。
`expose.clusterIP.annotations`:: 设置ClusterIP服务注释。默认值为 `{}`。
`expose.clusterIP.ports.httpPort`:: 设置HTTP服务端口。默认值为 `80`。
`expose.clusterIP.ports.httpsPort`:: 设置 HTTPS 服务端口。默认值为 `443`。
`expose.clusterIP.labels`:: 设置 ClusterIP 特定标签。默认值为 `{}`。
`expose.nodePort.name`:: 设置 NodePort 服务名称。默认值为 `harbor`。
`expose.nodePort.ports.http.port`:: 设置HTTP服务端口。默认值为 `80`。
`expose.nodePort.ports.http.nodePort`:: 设置 HTTP 节点端口。默认值为 `30002`。
`expose.nodePort.ports.https.port`:: 设置 HTTPS 服务端口。默认值为 `443`。
`expose.nodePort.ports.https.nodePort`:: 设置 HTTPS 节点端口。默认值为 `30003`。
`expose.nodePort.annotations`:: 设置 NodePort 注释。
`expose.nodePort.labels`:: 设置 NodePort 特定标签。默认值为 `{}`。
`expose.loadBalancer.name`:: 设置服务名称。默认值为 `harbor`。
`expose.loadBalancer.IP`:: 在支持 IP 分配时设置 loadBalancer IP。默认值为 `""`。
`expose.loadBalancer.ports.httpPort`:: 设置HTTP服务端口。默认值为 `80`。
`expose.loadBalancer.ports.httpsPort`:: 设置 HTTPS 服务端口。默认值为 `30002`。
`expose.loadBalancer.annotations`:: 设置 loadBalancer 服务注释。默认值为 `{}`。
`expose.loadBalancer.labels`:: 设置 loadBalancer 特定标签。默认值为 `{}`。
`expose.loadBalancer.sourceRanges`:: 指定 loadBalancerSourceRanges 的 IP 地址范围。默认值为 `[]`。

`persistence.persistentVolumeClaim.redis.annotations`:: 卷的注释。

`persistence.persistentVolumeClaim.trivy.storageClass`:: 提供卷的 `storageClass`。如果未指定，则使用默认的 StorageClass。

`registry.podAnnotations`:: 要添加到注册表 pod 的注释。默认值为 `{}`。
`registry.priorityClassName`:: pod 执行的优先级类。
`registry.secret`:: 在客户端和注册表存储后端之间保护上传状态的密钥。
`registry.credentials.username`:: {harbor} 核心的内部注册表访问用户名。默认值为 `harbor_registry_user`。
`registry.credentials.password`:: {harbor} 核心的内部注册表访问密码。默认值为 `harbor_registry_password`。
`registry.credentials.existingSecret`:: 包含 htpasswd 认证模式下注册表实例访问密码的现有密钥。默认值为 `""`。
`registry.credentials.htpasswdString`:: htpasswd 字符串格式的登录名和密码。排除 `registry.credentials.username` 和 `registry.credentials.password`。默认值为 `undefined`。
`registry.relativeurls`:: 当为真时，在 Location 头中返回相对 URL。如果 {harbor} 在反向代理后面，则为必需。默认值为 `false`。
`registry.upload_purging.enabled`:: 启用清除上传目录。默认值为 `true`。
`registry.upload_purging.age`:: 上传目录中文件被删除的时间段，默认是一周。默认值为 `168h`。
`registry.upload_purging.interval`:: 清除操作之间的时间间隔。默认值为 `24h`。
`registry.upload_purging.dryrun`:: 启用上传清除的干运行模式。默认值为 `false`。
`registry.initContainers`:: 在控制器的容器启动之前运行的初始化容器。默认值为 `[]`。


.{trivy} 参数
`trivy.enabled`:: 启用或禁用 {trivy} 扫描器。默认值为 `true`。
`trivy.image.repository`:: {trivy} 适配器镜像的储存库。默认值为 `private-registry/harbor-trivy-adapter`。
`trivy.image.tag`:: {trivy} 适配器镜像的标签。默认值为 `2.11`。
`trivy.resources`:: 为 {trivy} 适配器容器分配的资源。默认值为 `undefined`。
`trivy.automountServiceAccountToken`:: 是否挂载服务账户词元。默认值为 `false`。
`trivy.replicas`:: Pod 副本的数量。默认值为 `1`。
`trivy.debugMode`:: 启用 {trivy} 调试模式以进行故障排除。默认值为 `false`。
`trivy.vulnType`:: 以逗号分隔的漏洞类型列表（`os` 和 `library`）。默认值为 `os,library`。
`trivy.severity`:: 以逗号分隔的漏洞严重性列表以进行检查。默认值为 `UNKNOWN,LOW,MEDIUM,HIGH,CRITICAL`。
`trivy.ignoreUnfixed`:: 仅显示已修复的漏洞。默认值为 `false`。
`trivy.insecure`:: 跳过注册表证书验证。默认值为 `false`。
`trivy.skipUpdate`:: 禁用从 GitHub 下载 {trivy} 数据库。默认值为 `false`。
`trivy.skipJavaDBUpdate`:: 启用时需要手动下载 `trivy-java.db` 文件。默认值为 `false`。
`trivy.offlineScan`:: 防止 {trivy} 发送 API 请求以识别依赖项。默认值为 `false`。
`trivy.securityCheck`:: 逗号分隔的安全问题列表。默认值为 `vuln`。
`trivy.timeout`:: 等待扫描完成的持续时间。默认值为 `5m0s`。
`trivy.gitHubToken`:: 下载数据库所需的 GitHub 访问词元。默认值为 `undefined`。
`trivy.priorityClassName`:: 运行 pod 的优先级类别。默认值为 `undefined`。
`trivy.topologySpreadConstraints`:: 定义跨故障域的 pod 分配约束。默认值为 `undefined`。
`trivy.initContainers`:: 在主容器启动之前运行的初始化容器列表。默认值为 `[]`。

.数据库参数
`database.type`:: 数据库类型。使用外部数据库时设置为 `external`。默认值为 `internal`。
`database.internal.image.repository`:: 数据库镜像的储存库。默认值为 `private-registry/harbor-db`。
`database.internal.image.tag`:: 数据库镜像的标签。默认值为 `2.11`。
`database.internal.password`:: 内部数据库的密码。默认值为 `changeit`。
`database.internal.shmSizeLimit`:: {postgresql} 的共享内存大小限制（通常为容器内存限制的 50%）。默认值为 `512Mi`。
`database.internal.resources`:: 分配给数据库容器的资源。默认值为 `undefined`。
`database.internal.automountServiceAccountToken`:: 控制服务账户词元是否被挂载。默认值为 `false`。
`database.internal.initContainer.migrator.resources`:: 分配给数据库迁移初始化容器的资源。默认值为 `undefined`。
`database.internal.initContainer.permissions.resources`:: 分配给数据库权限初始化容器的资源。默认值为 `undefined`。
`database.internal.nodeSelector`:: 用于 Pod 分配的节点标签。默认值为 `{}`。
`database.internal.tolerations`:: 用于 Pod 分配的容忍度。默认值为 `[]`。
`database.internal.affinity`:: 节点或 pod 亲和性设置。默认值为 `{}`。
`database.internal.priorityClassName`:: 运行 pod 的优先级类别。默认值为 `undefined`。
`database.internal.livenessProbe.timeoutSeconds`:: 活跃性探针的超时时间（秒）：1-5 秒。默认值为 `1`。
`database.internal.readinessProbe.timeoutSeconds`:: 就绪探针的超时时间（秒）（范围：1-5 秒。默认值为 `1`。
`database.internal.extrInitContainers`:: 在数据库容器启动之前运行的附加初始化容器。默认值为 `[]`。
`database.external.host`:: 外部数据库的主机名。默认值为 `192.168.0.1`。
`database.external.port`:: 外部数据库的端口号。默认值为 `5432`。
`database.external.username`:: 外部数据库的用户名。默认值为 `user`。
`database.external.password`:: 外部数据库的密码。默认值为 `password`。
`database.external.coreDatabase`:: 核心服务使用的数据库名称。默认值为 `registry`。
`database.external.existingSecret`:: 包含数据库密码的现有密钥。密钥必须是 `password`。默认值为 `""`。
`database.external.sslmode`:: 外部数据库的连接方式。选项：`require`，`verify-full`，`verify-ca`，`disable`。默认值为 `disable`。
`database.maxIdleConns`:: 连接池中空闲连接的最大数量（0或更少表示不保留空闲连接）。默认值为 `50`。
`database.maxOpenConns`:: 数据库的最大打开连接数（0或更少表示无限制）。默认值为 `100`。
`database.podAnnotations`:: 要添加到数据库 Pod 的注释。默认值为 `{}`。

.跟踪参数
`trace.enabled`:: 启用跟踪功能。默认值为 `false`。
`trace.provider`:: 跟踪提供者（`jaeger` 或 `otel`）。Jaeger 版本应为 1.26 及以上。默认值为 `jaeger`。
`trace.sample_rate`:: 跟踪数据的采样率。`1` 采样 100%，`0.5` 采样 50%。默认值为 `1`。
`trace.namespace`:: 用于区分不同 {harbor} 服务的名称空间。
`trace.attributes`:: 用于跟踪提供者初始化的用户定义属性的键值字典。
`trace.jaeger.endpoint`:: Jaeger 跟踪的端点。默认值为 `\http://hostname:14268/api/traces`。
`trace.jaeger.username`:: Jaeger 认证的用户名。
`trace.jaeger.password`:: Jaeger 认证的密码。
`trace.jaeger.agent_host`:: Jaeger 的代理主机。
`trace.jaeger.agent_port`:: Jaeger 的代理端口。默认值为 `6831`。
`trace.otel.endpoint`:: {otelemetry} 跟踪的端点。默认值为 `hostname:4318`。
`trace.otel.url_path`:: {otelemetry} 的 URL 路径。默认值为 `/v1/traces`。
`trace.otel.compression`:: 为 {otelemetry} 启用压缩。默认值为 `false`。
`trace.otel.insecure`:: 为 {otelemetry} 建立不安全的连接。默认值为 `true`。
`trace.otel.timeout`:: {otelemetry} 的超时时间（秒）。默认值为 `10`。

.缓存参数
